## 🚀 프로젝트 개요
이 프로젝트는 유저가 설정한 여러 매개변수를 토대로 랜덤한 3D 행성을 생성, 실시간으로 탐험하고 관찰할 수 있게 하는 시뮬레이션 게임 개발을 목표로 합니다.
유저는 각 매개변수의 수치를 자유롭게 변경할 수 있습니다.

유저가 노이즈 함수, 지형 생성 알고리즘 등 전반적인 생성의 핵심 매개변수를 직접 조작하고, 그 결과가 행성의 지형, 생태계 등에 실시간으로 미치는 영향을 눈으로 확인하며 관련 원리를 쉽고 재미있게 학습하고 탐구할 수 있도록 돕습니다.
수많은 조합의 인풋값을 통해 독창적인 행성을 생성하여, 유저가 자신만의 고유한 행성을 발견하고 탐험하는 단순하고 캐주얼한 즐거움을 제공하는 것을 주된 목적으로 합니다.

### 🔑 주요 기능
- 매개변수 기반 행성 생성: 생성 번호(행성 코드), 지형크기 등 다양한 인풋값을 설정하여 행성의 형태를 결정합니다.
- 3D 랜더링 결과물 관찰: 생성된 행성을 3D 환경에서 자유롭게 회전 및 확대/축소하여 관찰합니다.

## 🛠 개발 중 어려웠던 점 & 해결 방법

### 1) ❗ 어려웠던 점
- 수업 중 배운 C++ 과 JS 지식 만으로는 따라가기 매우 벅찼다. 복잡한 알고리즘 구조와 처음 들어보는 WASM을 포함하여 다양한 난관이 있어 우리 조가 만들려는게 무엇이고 그 중 내가 무엇을 해야 하는지를 이해하는 것부터 어려웠다.
- 웹 구현을 위해 HTML과 CSS을 이용해야 했는데, 이 부분에 대해서는 어떠한 교육도 받지 못하고 살면서 처음 접해보는 부분이라 매우 어려웠다.
AI를 적극적으로 사용해도 된다는 조건이 있었으나, 애초에 AI가 짠 코드를 이해하기도, 그 코드에서 잘못된 부분을 찾는 것도 어려웠기에 이 부분에 투자한 시간이 압도적으로 많아 아쉬웠다.
- three.js 와 C++, WASM 모두 처음 접해보는 거라 적응하는데 어려웠다.
처음에 프로젝트 실행 구조를 정할 때 어떤 식으로 js에서 C++을 호출해야할지 헷갈려서 힘들었다.
- WASM에 대한 이해도가 부족하여 C++, WASM, JS의 상관관계에 대해 어려움을 겪었다.

### ✔ 해결 방법
- 튜터링, AI, 웹서핑 등 다양한 방법을 통해 어려운 부분들을 이해하기 위해 노력했다.
- AI를 통해 코드를 짜고, 공부하고, 이해하는 것 외에 방법이 없다고 느꼈다.
짧은 기간임을 감안하여 '완성만 해보자'라는 마음으로 빠르게 돌파 가능한 허들부터 하나씩 해결해나갔다.
- AI와 팀원들과 많은 대화를 나누어 어떤 식으로 구조를 잡아야할지 정해졌다.
- C++, WASM에 대한 관계성을 정보 수집하고 작업하며 이해하고 C++에 맞게 적용하였다.
